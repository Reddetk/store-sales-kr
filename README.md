## Описание

Репозиторий содержит программный код и вспомогательные материалы курсовой работы.  
Объект исследования — временные ряды дневных продаж розничной сети Corporación Favorita (Эквадор, 2013–2017).  
Предмет исследования — сравнительная точность шести прогнозных моделей на горизонтах 1, 3, 6 и 12 недель.

Датасет: [Store Sales — Time Series Forecasting (Kaggle)](https://www.kaggle.com/competitions/store-sales-time-series-forecasting)  
Состав: `train.csv` (~3 млн строк), `oil.csv`, `holidays_events.csv`, `stores.csv`, `transactions.csv`  
Период наблюдений: 2013–2017 | 54 магазина × 33 категории товаров

### Финальные модели

| Группа | Модель |
|---|---|
| МО | XGBoost, Random Forest, LSTM, Elastic Net |
| Эконометрика | SARIMA(p,d,q)(P,D,Q)×52, Holt-Winters (ETS) |

Метрики оценки: среднеквадратическая ошибка (RMSE), средняя абсолютная ошибка (MAE),  
средняя абсолютная процентная ошибка (MAPE). Целевое значение: MAPE ≤ 15 %.

## Описание файлов

### `data/`

| Путь | Содержимое |
|---|---|
| `data/raw/` | Исходные файлы Kaggle: `train.csv`, `oil.csv`, `holidays_events.csv`, `stores.csv`, `transactions.csv`. Файлы не изменяются на протяжении всего проекта. |
| `data/interim/` | Результат первичной очистки: заполнение пропусков в `oil.csv` методом прямого заполнения, удаление дублей, слияние таблиц по ключам `date`, `store_nbr`, `family`. |
| `data/processed/` | Финальный датасет в недельной агрегации с признаковым пространством: лаги, скользящие статистики, циклические кодировки дат, бинарные индикаторы праздников. |

### `models/saved/`

Папка предназначена для сохранения обученных моделей между сессиями Jupyter.  
Формат сохранения: XGBoost, Random Forest, Elastic Net — `.pkl` (через `joblib`);  
LSTM — `.h5` (через `model.save()`); SARIMA, Holt-Winters — `.pkl` (через `pickle`).

### `src/`

#### `src/config.py`
Центральный файл констант проекта: пути к папкам `data/`, `models/saved/`, `reports/`; горизонты прогнозирования `[1, 3, 6, 12]`; сезонный период `S = 52`; параметры разбивки обучающей и тестовой выборок.

#### `src/io/load_data.py`
Функции загрузки исходных CSV-файлов через `pandas`. Поддерживает кэширование промежуточных результатов в `data/interim/` для ускорения повторных запусков.

#### `src/io/preprocess.py`
Процедуры очистки: заполнение пропусков цены нефти методом прямого заполнения (`ffill`), удаление дублирующихся строк, слияние таблиц по ключам `date` и `store_nbr`, недельная агрегация целевой переменной `sales` функцией `sum`.

#### `src/features/build_features.py`
Конструирование признакового пространства: лаговые признаки (`lag_1`, `lag_2`, `lag_4`, `lag_12`, `lag_52`), скользящие средние и стандартные отклонения (`rolling_mean_4`, `rolling_std_4`, `rolling_mean_12`), циклическое кодирование дат через sin- и cos-преобразования, бинарные индикаторы праздников, one-hot encoding `store_type` и `cluster`.

#### `src/models/ml_models.py`
Классы-обёртки для XGBoost, Random Forest и Elastic Net на основе интерфейса `sklearn`. Каждый класс реализует методы `fit`, `predict` и `save` с сохранением в `models/saved/` через `joblib`.

#### `src/models/lstm_model.py`
Построение и обучение нейронной сети с долгой краткосрочной памятью (LSTM) средствами TensorFlow/Keras. Содержит функцию преобразования двумерного датасета в трёхмерные тензоры формата `(samples, timesteps, features)`, настройку early stopping и сохранение весов модели.

#### `src/models/sarima_model.py`
Функции идентификации, оценки и прогнозирования сезонной авторегрессионной интегрированной модели скользящего среднего (SARIMA) средствами `statsmodels`. Включает автоматический подбор порядков через `pmdarima.auto_arima` и проверку остатков тестом Льюнга–Бокса.

#### `src/models/ets_model.py`
Реализация модели экспоненциального сглаживания Хольта–Уинтерса (`ExponentialSmoothing` из `statsmodels`). Подбор параметров alpha, beta, gamma выполняется через `scipy.optimize.minimize`.

#### `src/models/tuning.py`
Вспомогательные процедуры подбора гиперпараметров: `GridSearchCV` для XGBoost и Elastic Net, `RandomizedSearchCV` для Random Forest, интеграция с Optuna для байесовской оптимизации XGBoost.

#### `src/evaluation/metrics.py`
Функции расчёта RMSE, MAE и MAPE. Принимают на вход массивы фактических и прогнозных значений, возвращают словарь метрик. Поддерживают вычисление по нескольким горизонтам за один вызов.

#### `src/evaluation/backtesting.py`
Реализация кросс-валидации на скользящем окне (walk-forward CV): итерационное расширение обучающей выборки с фиксированным шагом, сбор метрик по каждому фолду и горизонту прогнозирования.

#### `src/evaluation/plots.py`
Функции построения графиков средствами `matplotlib` и `seaborn`: динамика продаж, STL-декомпозиция, АКФ/ЧАКФ, прогноз против факта, распределение остатков. Все функции принимают параметр `save_path` для автоматического сохранения в `reports/figures/`.

#### `src/evaluation/export.py`
Процедуры экспорта итоговых таблиц метрик в форматы CSV и XLSX для последующей вставки в пояснительную записку. Формирует файлы в `reports/tables/` с именами, соответствующими номерам таблиц ПЗ (например, `table_3_10_metrics.xlsx`).

#### `src/explainability/shap_utils.py`
Расчёт SHAP-значений через `shap.TreeExplainer` для древесных моделей. Содержит функции построения summary plot, bar plot и dependence plot с автоматическим сохранением в `reports/figures/`.
